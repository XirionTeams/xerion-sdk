# Governance

## Contribution Rules

- Security > features.
- No breaking changes without a major version.
- Every feature must include tests and documentation updates.
- Minimum two senior reviewers required.
- Maintainer-only releases.

## Release Process

- Changelog required.
- Signed tags required.
- Dependency audit required.
- Reproducible build verification required.

## Support Policy

- Long-term maintenance with stable APIs.
- Deprecations are documented and versioned.
